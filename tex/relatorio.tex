\section*{Descrição do Projeto}
Este projeto apresenta a implementação de um sistema de reserva de salas, desenvolvido como parte da primeira atividade de Estrutura de Dados. O objetivo central é gerenciar a ocupação acadêmica de forma eficiente, respeitando restrições de horários e capacidade física.

O sistema permite as seguintes operações fundamentais:
\begin{itemize}
    \item \textbf{Reserva de salas:} Alocação de disciplinas com base em dia, horário e demanda de alunos.
    \item \textbf{Cancelamento:} Remoção de reservas existentes no sistema.
    \item \textbf{Visualização:} Exibição da grade de horários completa de cada sala.
\end{itemize}

Comando de compilação:
\begin{verbatim}
    g++ main.cpp ReservationSystem.cpp ReservationRequest.cpp 
    -o Reservation_main_test.out 
    -std=c++23 -Wall -Wextra -Wconversion -Wpedantic -O2
\end{verbatim}
\section*{Organização do Sistema}
O projeto foi modularizado em classes para garantir a separação de responsabilidades e facilitar a manutenção:

\begin{itemize}
    \item \texttt{main.cpp}: Ponto de entrada que executa os testes automatizados e demonstra as funcionalidades.
    \item \texttt{ReservationRequest}: Define a estrutura da solicitação de reserva (Arquivos \texttt{.cpp} e \texttt{.hpp}).
    \item \texttt{ReservationSystem}: Núcleo do sistema, responsável pelo gerenciamento dos dados e lógica de reserva (Arquivos \texttt{.cpp} e \texttt{.hpp}).
\end{itemize}

\section*{Estruturas de Dados e Implementação}
Para cumprir os requisitos de não utilizar estruturas da \textit{Standard Template Library} (STL), o sistema baseia-se em alocação dinâmica de memória:

\begin{enumerate}
    \item \textbf{Matriz de Disponibilidade:} Cada sala armazena uma matriz bidimensional estática $[dia][horario]$ (5 dias x 14 horários). Isso permite que a verificação de disponibilidade em um slot específico seja realizada em tempo constante $O(1)$.
    \item \textbf{Alocação Dinâmica:} As salas são organizadas em um array alocado dinamicamente. A busca por uma sala disponível para uma solicitação específica possui complexidade assintótica $O(n)$, onde $n$ é o número de salas.
    \item \textbf{Melhor Ajuste:} O sistema busca a sala com a menor capacidade que comporte o número de alunos da disciplina. Essa estratégia otimiza o uso do espaço, preservando salas maiores para demandas que realmente necessitem de tal capacidade.
\end{enumerate}

\section*{Análise Crítica}
A implementação atual prioriza a velocidade de acesso e a simplicidade de implementação. Contudo, observa-se um custo de memória fixo, dado que cada sala reserva espaço para todos os horários possíveis, independentemente de estarem ocupados ou não.

Em cenários de alta escalabilidade (milhares de salas e horários), essa abordagem pode se tornar ineficiente. Uma solução proposta para versões futuras é a utilização de listas encadeadas para armazenar apenas as reservas ativas, substituindo a matriz densa por uma estrutura esparsa, otimizando assim o consumo de memória.
